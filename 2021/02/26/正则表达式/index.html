正则表达式
2021-02-26 19:19:52 # java #正则表达式

标准字符集合

字符
\d 任意一个数字，0-9中任意一个
\D 任意非数字
\w 任意一个字母、下划线、数字。即：A-Z a-z 0-9
\W 表示除[0－9a-zA-Z_]之外的字符
\s 匹配任何空白字符，包括空格、制表符、换页符等等。等价于 [ \f\n\r\t\v]
\S 匹配任何非空白字符。
. 匹配任意一个字符
如要匹配包括\n在内的所有字符一般用：[\s \S]
\b 单词边界
\B 匹配非单词边界

自定义字符集

  • s使用中括号[] 包裹

  • 特殊字符被包含到中括号中失去特殊意义，除了^,-之外。

  • 标准字符集合中除小数字外，包括在中括号中，自定义字符集合将包括该集合。

量词

使用 意义
{n} n次
{n,m} 至少n次，至多m次
{n,} 至少n次
？ 0或1次。相当于{0,1}
+ 至少1次。相当于{1,}
* 0或任意次。相当于：{0,}

贪婪模式：匹配字符越多越好(默认)

非贪婪模式：匹配字符越少越好

模式设置：在修饰匹配次数后加一个?即为非贪婪模式

字符边界

该组标记匹配的不是字符而是位置

符号 含义
^ 以……开始
$ 以……结束
\b 匹配一个单词边界

\b 匹配：前面字符和后面字符不全是\w

正则表达式匹配模式

模式
IGNORECASE 忽略大小写
SINGLELINE 单行模式。整个文本看作一个字符串(小数点.可匹配\n在内任意字符)
MULTILINE 多行模式。每行都是一个字符串
如需仅匹配开始和结束位置，可用\A 和 \Z

选择符和分组

| ：或关系，匹配|左边或右边

**()**：捕获组

  • 括号中表达式可做整体修饰

  • 取匹配结果时，括号中表达式匹配到的内容可单独得到。获取：\1

  • 每一个括号分配一个编号，由左括号顺序从1开始自动编号。

    • 捕获是由整个正则表达式模式匹配的文本

**(?:Expression)**：非捕获组

  • 一些表达式中不得不使用(),但又不需要取保存()中子表达式匹配到的内容，这时可以用非捕获组来抵消使用()带来的副作用

反向引用

格式：\n

  • 每一对()会分配一个编号，根据左括号顺序从1开始
  • 通过反向引用，可对分组已捕获的字符串进行引用

预搜索

  • 只进行子表达式匹配，匹配内容不计入最终匹配结果，(即零宽度)

  • 判断当前位置前后字符，但不匹配前后字符

  • 匹配字符内容保存到最终结果为占有字符

    • 匹配字符位置匹配内容不保存到结果为零宽度

(?= Expression) ：断言自身出现位置后面能匹配表达式

**(?<= Expression)**：断言自身出现位置前面能匹配表达式

**(?! Expression)**：断言自身出现位置后面不能匹配表达式

**(?<! Expression)**：断言自身出现位置前面不能匹配表达式
